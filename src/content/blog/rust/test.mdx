## Rust 테스트 기본

<Callout type="info">
Rust는 별도의 테스트 프레임워크 없이도 언어 자체에 테스트 기능이 내장되어 있습니다.
</Callout>

`#[test]` 어트리뷰트를 함수에 붙이면 `cargo test`로 실행할 수 있습니다.

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn it_works() {
        let result = add(2, 2);
        assert_eq!(result, 4);
    }
}
```

## assert 매크로

Rust는 세 가지 주요 assert 매크로를 제공합니다:

| 매크로 | 용도 |
|--------|------|
| `assert!` | 조건이 true인지 확인 |
| `assert_eq!` | 두 값이 같은지 확인 |
| `assert_ne!` | 두 값이 다른지 확인 |

```rust
#[test]
fn greeting_contains_name() {
    let result = greeting("Carol");
    assert!(
        result.contains("Carol"),
        "인사말에 이름이 포함되어야 합니다. 실제 값: `{result}`"
    );
}
```

<Callout type="warning">
`assert_eq!`와 `assert_ne!`를 사용하려면 비교 대상 타입이 `PartialEq`와 `Debug` 트레이트를 구현해야 합니다.
</Callout>

## should_panic 테스트

패닉이 발생해야 하는 경우를 테스트할 수 있습니다:

```rust
#[test]
#[should_panic(expected = "100보다 클 수 없습니다")]
fn greater_than_100() {
    Guess::new(200);
}
```

## Result를 반환하는 테스트

`Result<T, E>`를 반환하면 `?` 연산자를 사용할 수 있어 편리합니다:

```rust
#[test]
fn it_works() -> Result<(), String> {
    let result = add(2, 2);
    if result == 4 {
        Ok(())
    } else {
        Err(format!("2 + 2는 4여야 하지만, 실제 값: {result}"))
    }
}
```

## 통합 테스트

<Callout type="success">
통합 테스트는 `tests/` 디렉토리에 별도 파일로 작성하며, 라이브러리의 공개 API만 사용하여 외부 사용자 관점에서 테스트합니다.
</Callout>

```rust
// tests/integration_test.rs
use my_crate;

#[test]
fn it_adds_two() {
    assert_eq!(4, my_crate::add_two(2));
}
```

각 파일이 독립적인 크레이트로 컴파일되므로, 모듈 간 공유 코드가 필요하면 `tests/common/mod.rs` 패턴을 사용합니다.
