## MemPalace란?

[MemPalace](https://github.com/MemPalace/mempalace)는 **AI 세션 간 대화와 결정 사항을 영구 보존하고 검색 가능하게 만드는 오픈소스 메모리 시스템**이다. Claude, ChatGPT, 로컬 LLM 어디든 연결할 수 있다.

핵심 문제의식은 단순하다. **AI와 나눈 중요한 대화, 아키텍처 결정, 디버깅 과정이 세션이 끝나면 사라진다.** 다음 세션에서 같은 설명을 반복해야 한다. 수개월에 걸쳐 쌓인 컨텍스트가 매번 리셋된다.

MemPalace의 해법은 **"절대 요약하지 말고, 그냥 전부 저장하라"**는 것이다. LLM이 요약할 때 발생하는 손실 없이, 원본 대화를 그대로 보존한다. 그리고 고대 기억술인 [기억의 궁전(Method of Loci)](https://en.wikipedia.org/wiki/Method_of_loci)에서 영감을 받은 계층 구조로 조직화해 언제든 꺼낼 수 있게 한다.

```
대화/결정 → MemPalace 저장 → ChromaDB 인덱싱 → 다음 세션에서 검색
                                                  (세션 경계 없음)
```

Python 3.9+로 작성되었고, MIT 라이선스로 공개되어 있다. LongMemEval 벤치마크에서 **96.6% R@5** 달성 — 현재까지 발표된 결과 중 최고치다.

## 왜 주목할 만한가?

AI와 장기 프로젝트를 진행할 때 가장 큰 문제는 **컨텍스트 소멸**이다. 세션 경계를 넘어 지식이 유지되지 않는다. 기존 해법들은 불완전하다.

| 방법 | 문제점 |
|---|---|
| 매번 컨텍스트 복사-붙여넣기 | 번거롭고, 컨텍스트 창 낭비 |
| LLM에게 요약 요청 | 정보 손실 발생, 요약 품질 불균일 |
| 노션/옵시디언 수동 정리 | AI가 직접 읽을 수 없음 |
| RAG 시스템 직접 구축 | 개발 비용 높고, 조직화 어려움 |

MemPalace는 이 틈새를 채운다.

| 항목 | 내용 |
|---|---|
| 라이선스 | MIT |
| 언어 | Python 3.9+ |
| 벡터 DB | ChromaDB |
| 통합 | MCP (29개 도구), Claude Code 훅 |
| 비용 | 완전 무료, 클라우드 불필요 |
| 벤치마크 | LongMemEval 96.6% R@5 |

## 아키텍처: 기억의 궁전 구조

MemPalace는 고대 기억술을 그대로 소프트웨어 구조로 구현했다. 계층 구조를 따라 기억을 조직화하고, 시맨틱 검색으로 회수한다.

```
Palace (최상위)
├── Wing (인물/프로젝트별 공간)
│   ├── Room (토픽 카테고리)
│   │   ├── Hall (기억 유형)
│   │   │   ├── Closet (요약 → 원본 포인터)
│   │   │   └── Drawer (원본 텍스트 verbatim 저장)
│   │   └── Hall ...
│   └── Room ...
└── Tunnel (Wing 간 교차 연결)
```

### 각 레이어의 역할

**Wing**: 사람 또는 프로젝트 단위. `leeyc-blog`, `payment-service`, `alice` 같은 단위로 구분된다.

**Room**: 토픽 카테고리. `auth`, `billing`, `deployment`, `architecture` 등.

**Hall**: 기억의 유형을 분류한다. 사실(facts), 이벤트(events), 발견(discoveries), 선호도(preferences), 조언(advice) 등.

**Drawer**: 원본 텍스트가 저장되는 최소 단위. LLM 요약 없이 **verbatim**(그대로) 저장한다.

**Closet**: Drawer들을 가리키는 요약 색인. 원본으로 접근하는 포인터 역할.

**Tunnel**: 다른 Wing의 관련 기억을 연결하는 크로스 레퍼런스. 예를 들어 `payment-service`의 인증 로직이 `auth-library` Wing의 특정 결정과 연결된다.

## 저장 방식: 손실 없는 보존

MemPalace의 가장 중요한 설계 원칙은 **"절대 요약하지 않는다"**는 것이다.

```
일반적인 AI 메모리 시스템:
원본 대화 → LLM 요약 → 요약본 저장
              (정보 손실 발생)

MemPalace:
원본 대화 → ChromaDB + 계층 구조 저장
              (손실 없음)
```

ChromaDB에 원본 텍스트를 벡터로 임베딩해서 저장한다. 검색할 때는 시맨틱 유사도로 관련 기억을 찾는다. Closet은 Drawer의 포인터 역할만 하고, 원본은 항상 Drawer에 온전히 보존된다.

### 필터링으로 정밀도 향상

단순 벡터 검색 대비 Wing + Room 필터를 조합하면 정밀도가 크게 오른다.

| 검색 방식 | 정확도 |
|---|---|
| 필터 없는 전체 검색 | 기준선 |
| Wing + Room 필터 적용 | **R@10 기준 34% 향상** |

## 지식 그래프: 시간 축이 있는 관계망

단순 텍스트 저장을 넘어, MemPalace는 **시간 축이 있는 엔티티-관계 지식 그래프**를 SQLite로 유지한다.

```
(엔티티A) -[관계]-> (엔티티B) [유효기간: 2025-01-01 ~ 2025-06-30]
```

예를 들어 "프로젝트 X는 PostgreSQL을 사용한다"는 사실을 저장할 때, 언제부터 언제까지 유효한 사실인지를 함께 기록한다. 나중에 MySQL로 마이그레이션했을 때, 기존 사실이 삭제되는 게 아니라 **유효기간이 닫히고 새 사실이 추가**된다. 변경 이력이 자연스럽게 보존된다.

## MCP 통합: 29개 도구로 Claude와 연결

Claude, Claude Code와의 통합은 **MCP(Model Context Protocol) 서버**를 통해 이루어진다. 29개의 도구를 제공한다.

```bash
# MCP 서버 실행
mempalace serve --mcp
```

Claude가 대화 중에 직접 기억을 저장하고 검색할 수 있다.

```
Claude: "지난번에 GraphQL로 전환한 이유가 뭐였죠?"
→ MCP 도구 호출: search_memories("why switch GraphQL")
→ 관련 Drawer 반환
→ Claude가 맥락을 이어서 답변
```

Claude Code에서는 **자동 저장 훅**을 설정할 수 있다. 개발 세션이 끝날 때 중요한 결정사항이 자동으로 팔래스에 기록된다.

## AAAK: 토큰 압축 실험

MemPalace는 **AAAK 다이얼렉트**라는 실험적 손실 압축 방식을 개발 중이다. 엔티티 코드와 문장 단축으로 토큰을 줄이는 방식이다.

| 방식 | LongMemEval 정확도 |
|---|---|
| 압축 없음 (원본) | **96.6%** |
| AAAK 압축 | 84.2% |

개발팀은 AAAK가 현재 정확도를 낮춘다는 사실을 투명하게 공개하고 있다. "정확도를 희생하면서까지 토큰을 아끼는 게 맞는가"를 계속 검토 중이다. LLM이 별도 디코딩 없이 직접 읽을 수 있다는 것이 특징이다.

## 설치 및 사용

```bash
pip install mempalace

# 팔래스 초기화
mempalace init ~/projects/myapp

# 기존 대화 파일 마이닝
mempalace mine ~/chats/ --mode convos

# 검색
mempalace search "왜 GraphQL로 전환했나"

# MCP 서버 실행 (Claude 연동)
mempalace serve --mcp
```

Python API로도 접근할 수 있어서, 기존 워크플로우에 임베딩하거나 커스텀 에이전트에 통합하기 쉽다.

## 알려진 이슈

프로젝트에는 현재 열려 있는 이슈들이 있다. 개발팀이 직접 문서화해두었다.

- **데이터 저장 실패**: `mempalace_add_drawer`가 성공으로 보고하지만 실제로 저장되지 않는 케이스
- **보안 취약점**: 데이터 누출, 쉘 인젝션, 파일 권한 문제
- **상태 카운트 오류**: `status` 명령이 10,000개 이상의 Drawer를 있을 때 잘못 카운트
- **ChromaDB 동시성**: SQLite 기반 ChromaDB의 동시 접근 이슈

활발히 개발 중인 프로젝트답게 이슈가 있지만, 개발팀이 투명하게 공개하고 수정하고 있다는 점은 긍정적이다. 2026년 4월에는 이전에 과장된 수치들을 직접 정정하는 공지를 올리기도 했다.

## 마무리

MemPalace의 핵심 통찰은 **"AI 메모리 문제는 요약의 문제가 아니라 조직화와 검색의 문제"**라는 것이다. LLM에게 기억을 요약시키면 정보가 손실된다. 원본을 그대로 보존하고, 잘 조직화해서, 필요할 때 꺼내는 것이 맞다.

Palace 구조는 이 목표에 적합하다. 계층 구조가 있으니 좁은 범위로 필터링해서 정밀도를 높일 수 있다. Tunnel로 크로스 레퍼런스를 만들 수 있어서 연관된 프로젝트 간 지식이 연결된다. 시간 축이 있는 지식 그래프라 "당시에는 맞았던 결정"과 "지금 유효한 사실"을 구분할 수 있다.

완성도 면에서 아직 이슈가 있고, AAAK 압축은 트레이드오프가 명확하지 않은 단계다. 하지만 **장기 프로젝트에서 Claude와 협업할 때 컨텍스트 유실 문제**를 겪고 있다면, 지금 당장 써볼 수 있는 가장 체계적인 해법 중 하나다. 완전 로컬, 완전 무료, MCP로 Claude와 즉시 연동 가능하다.
